# 项目中文代码阅读说明

这份文档是配合代码内联注释一起看的。

适合两种场景：
- 你第一次通读项目，不想在很多文件之间反复跳转
- 你已经看到某个变量名，但不知道它是“整图尺度”还是“patch 尺度”

## 全局变量约定

- `data`：通常表示输入图像的整图数据，形状常见为 `[H, W, C]`
- `aux_data`：第二模态整图数据，形状常见为 `[H, W, C2]`
- `gt`：整图真实标签，形状为 `[H, W]`
- `Train_Label / Test_Label / Val_Label`：整图尺度的训练、测试、验证标签掩码
- `Train_Split_* / Test_Split_*`：切成 patch 后的数据、标签、伪标签
- `sp_gt / SP_gt / SP_label / ERS_label`：超像素标签图
- `pseudo_labels / pseudo_probs / clip_probs`：伪标签概率图或由概率取 argmax 得到的标签
- `patch_height / patch_width`：每个 patch 去掉 `EDGE` 后的有效尺寸

## 运行主线

### [seg_main.py](/D:/Master_medium/HZSCM/seg_main.py)

主流程可以按 4 步理解：

1. `read_data()` 读入整图、伪标签和 patch
2. 第一阶段只用真实标签预训练
3. 用预训练结果修正伪标签
4. 第二阶段用真实标签和修正后的伪标签继续训练

关键变量：

- `args`：命令行配置对象
- `del_class`：被拿掉监督、当作 unseen 的类别编号，从 0 开始
- `AC / OA / AA / KA`：最终实验的各类精度统计
- `PRE_AC / PRE_OA / PRE_AA / PRE_KA`：预训练阶段的评估结果
- `PreOutputWhole`：预训练模型测试后拼回整图的预测标签
- `pseudo_labels_c / pseudo_probs_c`：修正后的整图伪标签标签图和概率图
- `OutputWhole`：最终模型测试后拼回整图的预测标签

### [data_read.py](/D:/Master_medium/HZSCM/data_read.py)

它只是一个分发入口：

- `fusion_mode='single'`：调用单模态读数逻辑
- `fusion_mode='early'`：调用多模态早期融合读数逻辑

## 数据读取

### [data_read_singlemodal.py](/D:/Master_medium/HZSCM/data_read_singlemodal.py)

虽然叫“单模态”，但它现在支持：

- 训练时只使用主模态
- 伪标签生成时可选择主模态 + 第二模态共同参与

关键函数：

- `_load_multimodal_dataset()`：读取 Houston / Berlin / MUUFL 这样的双模态数据集
- `_load_sp_gt()`：读取某个模态对应的 ERS 超像素图
- `_load_sam_list()`：读取某个模态对应的 SAM mask 列表
- `_build_modality_images()`：把一个或两个模态都转成 RGB 图供 CLIP 使用
- `_load_or_generate_pseudo_labels()`：优先读缓存；没有缓存时在线生成伪标签
- `read_data()`：最后把整图标准化、划分训练测试、切成 patch

`_load_or_generate_pseudo_labels()` 中最重要的变量：

- `use_multimodal`：是否启用双模态伪标签
- `modality_images`：送给 CLIP 的 RGB 图列表
- `modality_sp_gts`：每个模态自己的 ERS 超像素图
- `modality_sam_lists`：每个模态自己的 SAM 掩码列表
- `clip_outputs`：不同 CLIP 权重得到的整图概率，最后再平均

`read_data()` 中最重要的变量：

- `class_count`：类别数
- `gt_reshape`：把二维标签图拉平成一维后的结果
- `train_rand_idx`：按类别保存的训练像素索引
- `train_data_index / test_data_index / val_data_index`：最终展开的一维索引集合

### [data_read_multimodal.py](/D:/Master_medium/HZSCM/data_read_multimodal.py)

它和单模态版最大的区别只有一件事：

- `fused_data = concat(primary, secondary)`

关键变量：

- `primary_data`：主模态整图
- `aux_data`：第二模态整图
- `_standardize_cube()`：每个模态分别做标准化
- `_fuse_modalities()`：沿通道维拼接两个模态
- `fused_data`：最终送进网络的融合输入

## 数据集封装

### [hyper_dataset.py](/D:/Master_medium/HZSCM/hyper_dataset.py)

- `self.data`：patch 图像
- `self.labels`：patch 真实标签
- `self.sp_labels / self.pl_labels`：patch 伪标签
- `self.scores`：可选的伪标签置信度

## 模型

### [seg_model.py](/D:/Master_medium/HZSCM/seg_model.py)

结构很直接：

- `HSI_UNet`：提取像素级特征
- `projection_head`：把像素特征映射成类别 logits

关键变量：

- `x`：图像或特征张量
- `out`：像素级 logits
- `loss_l`：真实标签损失
- `loss_u`：伪标签损失
- `conv0 / conv1 / conv2`：编码阶段保留下来的跳连特征

## 工具函数

### [util.py](/D:/Master_medium/HZSCM/util.py)

这是项目里最值得慢慢读的文件。

推荐阅读顺序：

1. `get_pseudo_labels()`
2. `fuse_SAM_ERS()`
3. `get_pseudo_labels_using_SAM_and_ERS()`
4. `SpiltHSI()`
5. `PatchStack()`
6. `get_sp_label()`
7. `get_cluster_labels()`
8. `local_aggregation()`
9. `correct_pseudo_labels()`

关键变量解释：

- `image_probs`：整图每个像素的类别概率
- `new_SAM_label / new_ERS_label`：SAM 和剩余 ERS 的融合标签图
- `sp_features`：每个超像素的平均特征
- `sp_labels`：每个超像素对应的监督标签，没标签记为 `-1`
- `cluster_index`：每个超像素所在的聚类编号
- `pseudo_probs_c`：在修正过程中被重新排列过类别概率的概率矩阵
- `whole_pseudo_labels / whole_pseudo_probs`：从超像素级重新回填到像素级的结果

## 分割结果生成

### [segment/segmentation_dataset_utils.py](/D:/Master_medium/HZSCM/segment/segmentation_dataset_utils.py)

主要做三类事情：

- 读不同数据集的主模态、第二模态和 GT
- 把任意模态转成 RGB 图
- 统一生成文件路径

重要函数：

- `available_modalities()`：判断当前数据集是单模态还是双模态
- `dataset_rgb_image()`：按模态返回 RGB 图
- `default_sam_output_path()`：返回默认 SAM 保存路径
- `default_ers_output_path()`：返回默认 ERS 保存路径

### [segment/generate_sam_masks.py](/D:/Master_medium/HZSCM/segment/generate_sam_masks.py)

默认会按数据集自动跑：

- 单模态数据集：只生成 `primary`
- 双模态数据集：生成 `primary + secondary`

关键变量：

- `modalities`：当前要处理的模态列表
- `mask_generator`：SAM 自动分割器
- `masks`：SAM 输出的 mask 列表

### [segment/generate_ers_superpixels.py](/D:/Master_medium/HZSCM/segment/generate_ers_superpixels.py)

关键变量：

- `backend_module`：本机已安装的 ERS 包
- `used_backend`：实际使用的后端名字
- `labels`：生成出的超像素标签图

## 分割结果可视化

### [segment/visualize/visualize_segmentations.py](/D:/Master_medium/HZSCM/segment/visualize/visualize_segmentations.py)

关键变量：

- `rgb_image`：当前模态的 RGB 图
- `ers_labels`：ERS 超像素标签图
- `sam_list`：SAM mask 列表
- `sam_labels`：把 SAM mask 列表转换成的二维标签图
- `ers_boundary / sam_boundary`：边界像素布尔掩码
- `ers_color / sam_color`：伪彩色分割图
- `ers_overlay / sam_overlay / both_overlay`：叠加边界后的图像

## 读代码建议

如果你准备继续改这个项目，推荐顺序是：

1. 先看 [config.py](/D:/Master_medium/HZSCM/config.py)
2. 再看 [seg_main.py](/D:/Master_medium/HZSCM/seg_main.py)
3. 然后对照 [data_read_singlemodal.py](/D:/Master_medium/HZSCM/data_read_singlemodal.py) / [data_read_multimodal.py](/D:/Master_medium/HZSCM/data_read_multimodal.py)
4. 最后重点啃 [util.py](/D:/Master_medium/HZSCM/util.py)

这样最容易把“数据怎么来、模型怎么训、伪标签怎么修”三条线串起来。
